# Git Blame

**Git Blame** ポップアップは、追跡されているファイルの行ごとの作成者情報を表示します。各行を誰が、いつ、どのコミットで変更したかを確認できます。

## Git Blame へのアクセス

任意のパネルから（ポップアップが開いていない状態で）`b` を押すと Blame ポップアップが開きます。

## キーバインディング

### ファイル選択ビュー

| キー           | アクション                       |
| :------------- | :------------------------------- |
| `↑` または `k` | ファイルリストを上に移動         |
| `↓` または `j` | ファイルリストを下に移動         |
| `enter`        | 選択したファイルの Blame を表示  |
| `esc`          | Blame ポップアップを閉じる       |

### Blame ビュー（ファイル選択後）

| キー           | アクション           |
| :------------- | :------------------- |
| `↑` または `k` | 上にスクロール       |
| `↓` または `j` | 下にスクロール       |
| `←` または `h` | 左にスクロール       |
| `→` または `l` | 右にスクロール       |
| `esc`          | ファイルリストに戻る |

## ワークフロー

### ファイルの Blame を表示

1.  **開く**: 任意のパネルから `b` を押します。
2.  **フィルター** (オプション): テキストを入力してファイルパスでリストを絞り込みます。
3.  **選択**: `↑/↓` で目的のファイルに移動します。
4.  **表示**: `enter` を押して Blame 出力を読み込みます。
5.  **スクロール**: 矢印キーで Blame 出力を操作します（長い行は水平スクロール対応）。
6.  **戻る**: `esc` でファイルリストに戻り、もう一度 `esc` でポップアップを閉じます。

:::info
Blame ポップアップには、変更されたファイルだけでなく、リポジトリ内のすべての Git 追跡ファイルが表示されます。現在の状態に関わらず、追跡ファイルであれば Blame を実行できます。
:::
