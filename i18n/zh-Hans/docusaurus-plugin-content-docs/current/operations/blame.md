# Git Blame

**Git Blame** 弹窗显示任意已追踪文件的逐行作者信息——每一行由谁、在什么时候、在哪个提交中修改。

## 访问 Git Blame

在任意面板中（无弹窗打开时）按 `b` 打开 Blame 弹窗。

## 键位绑定

### 文件选择视图

| 键             | 操作                       |
| :------------- | :------------------------- |
| `↑` 或 `k`    | 在文件列表中向上移动       |
| `↓` 或 `j`    | 在文件列表中向下移动       |
| `enter`        | 查看所选文件的 Blame 信息  |
| `esc`          | 关闭 Blame 弹窗            |

### Blame 视图（选择文件后）

| 键             | 操作             |
| :------------- | :--------------- |
| `↑` 或 `k`    | 向上滚动         |
| `↓` 或 `j`    | 向下滚动         |
| `←` 或 `h`    | 向左滚动         |
| `→` 或 `l`    | 向右滚动         |
| `esc`          | 返回文件列表     |

## 工作流程

### 查看文件的 Blame 信息

1.  **打开**: 在任意面板中按 `b`。
2.  **过滤** (可选): 输入文字按路径过滤文件列表。
3.  **选择**: 使用 `↑/↓` 定位到目标文件。
4.  **查看**: 按 `enter` 加载 Blame 输出。
5.  **滚动**: 使用方向键浏览 Blame 输出（支持横向滚动以查看长行）。
6.  **返回**: 按 `esc` 返回文件列表，再次按 `esc` 关闭弹窗。

:::info
Blame 弹窗列出仓库中所有 Git 追踪文件，而不仅仅是已修改的文件。无论文件当前状态如何，均可对任意已追踪文件执行 Blame。
:::
