# Git Blame

**Git Blame** 彈窗顯示任意已追蹤檔案的逐行作者資訊——每一行由誰、在什麼時候、在哪個提交中修改。

## 存取 Git Blame

在任意面板中（無彈窗開啟時）按 `b` 開啟 Blame 彈窗。

## 鍵位綁定

### 檔案選擇視圖

| 鍵             | 操作                       |
| :------------- | :------------------------- |
| `↑` 或 `k`    | 在檔案列表中向上移動       |
| `↓` 或 `j`    | 在檔案列表中向下移動       |
| `enter`        | 查看所選檔案的 Blame 資訊  |
| `esc`          | 關閉 Blame 彈窗            |

### Blame 視圖（選擇檔案後）

| 鍵             | 操作             |
| :------------- | :--------------- |
| `↑` 或 `k`    | 向上捲動         |
| `↓` 或 `j`    | 向下捲動         |
| `←` 或 `h`    | 向左捲動         |
| `→` 或 `l`    | 向右捲動         |
| `esc`          | 返回檔案列表     |

## 工作流程

### 查看檔案的 Blame 資訊

1.  **開啟**: 在任意面板中按 `b`。
2.  **篩選** (可選): 輸入文字按路徑篩選檔案列表。
3.  **選擇**: 使用 `↑/↓` 定位到目標檔案。
4.  **查看**: 按 `enter` 載入 Blame 輸出。
5.  **捲動**: 使用方向鍵瀏覽 Blame 輸出（支援橫向捲動以查看長行）。
6.  **返回**: 按 `esc` 返回檔案列表，再次按 `esc` 關閉彈窗。

:::info
Blame 彈窗列出儲存庫中所有 Git 追蹤檔案，而不僅僅是已修改的檔案。無論檔案當前狀態如何，均可對任意已追蹤檔案執行 Blame。
:::
